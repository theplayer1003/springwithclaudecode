# Spring 학습 프로젝트 — 게시판 → 대규모 서비스

## 학습자 정보

- 웹 백엔드 개발자 지망, 대규모 처리 시스템에 관심
- 자바 기초 문법을 읽을 수 있는 수준, 작은 콘솔 프로그램 3개 작성 경험
- 웹/프레임워크 지식은 없는 상태에서 시작
- 하루 5시간 이상 학습 가능

## 학습 방식

1. **개념 설명** — 주제별 md 파일을 생성하여 개념 설명. 학습자가 읽고 질문
2. **코딩** — 학습자가 직접 작성, 클로드는 힌트와 구조만 제시. 작성 후 클로드가 리뷰 → 수정 반복
3. **도전 과제** — 클로드가 과제 제시, 학습자가 풀고 리뷰 받음
4. **문서화** — 학습자가 직접 `docs/`에 정리 문서 작성 → 클로드가 리뷰 → 보완 (클로드가 대신 작성하지 않음)
5. **형상관리** — 커밋 → PR → merge → 브랜치 삭제. 학습자가 직접 Git 명령어 수행

### 세부 규칙

- **Q&A 기록** — 모든 과정에서 발생하는 질문/답변을 주제별 파일로 나누어 빠짐없이 즉시 기록 (예: `QA_Dirty_Checking.md`)
- **커밋 단위** — Phase 전체를 한 번에 커밋하지 않고, 의미 있는 단위로 나누어 커밋. 클로드가 적절한 시점을 판단하여 유도
- **코드/Git 주체** — 코드 작성과 Git 작업 모두 학습자가 직접 수행. 클로드는 힌트, 리뷰, 커밋 메시지 제안만 담당

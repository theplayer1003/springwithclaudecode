# 페이징 처리 (Pageable)

## 왜 페이징이 필요한가

게시글이 10,000개일 때 `findAll()`로 전부 가져오면:

- **서버** — 10,000개의 객체를 메모리에 올려야 한다
- **클라이언트** — 화면에 10,000개를 한 번에 보여줄 필요가 없다
- **네트워크** — 불필요하게 큰 응답 전송

해결: 한 번에 20개씩 잘라서 가져온다 (페이징).

## SQL의 LIMIT / OFFSET

```sql
-- 1페이지 (0~19번)
SELECT * FROM post ORDER BY created_at DESC LIMIT 20 OFFSET 0;

-- 2페이지 (20~39번)
SELECT * FROM post ORDER BY created_at DESC LIMIT 20 OFFSET 20;

-- 3페이지 (40~59번)
SELECT * FROM post ORDER BY created_at DESC LIMIT 20 OFFSET 40;
```

- `LIMIT` — 가져올 개수
- `OFFSET` — 건너뛸 개수

## Spring Data JPA의 Pageable

JPA에서는 LIMIT/OFFSET을 직접 쓰지 않고, `Pageable` 객체로 추상화하여 사용한다.

### Repository

```java
@Query("SELECT p FROM Post p JOIN FETCH p.member ORDER BY p.createdAt DESC")
Page<Post> findAllWithMember(Pageable pageable);
```

- 반환 타입: `Page<Post>` — 데이터 + 페이지 메타정보를 함께 담는 객체
- 파라미터: `Pageable` — 페이지 번호, 크기, 정렬 정보

### Service

```java
@Transactional(readOnly = true)
public Page<PostResponse> getAllPosts(Pageable pageable) {
    return postRepository.findAllWithMember(pageable)
            .map(PostResponse::from);
}
```

`Page`의 `.map()` 메서드로 `Page<Post>` → `Page<PostResponse>` 변환. Pageable은 분해하지 않고 그대로 전달한다.

### Controller

```java
@GetMapping
public ResponseEntity<Page<PostResponse>> getAllPosts(Pageable pageable) {
    return ResponseEntity.ok(postService.getAllPosts(pageable));
}
```

Spring이 쿼리 파라미터(`page`, `size`, `sort`)를 자동으로 `Pageable` 객체로 변환해준다.

### 클라이언트 요청 예시

```
GET /posts                       → 기본값 (page=0, size=20)
GET /posts?page=0&size=10        → 1페이지, 10개씩
GET /posts?page=2&size=5         → 3페이지, 5개씩
GET /posts?sort=title,desc       → 제목 내림차순
```

### Page 응답 JSON 구조

`Page` 객체는 단순 배열이 아니라 메타정보를 포함한 구조로 변환된다:

```json
{
  "content": [{"title": "...", "author": "..."}, ...],
  "totalPages": 50,
  "totalElements": 1000,
  "size": 20,
  "number": 0
}
```

클라이언트는 `totalPages`를 이용해 페이지 네비게이션 UI를 구성할 수 있다.

### 기본값 설정

`application.properties`에서 기본 page size를 설정할 수 있다:

```properties
spring.data.web.pageable.default-page-size=20
```

## Page 객체 생성 (테스트용)

테스트에서 `Page`를 모킹할 때는 구현체인 `PageImpl`을 사용한다:

```java
List<PostResponse> responses = List.of(...);
Page<PostResponse> page = new PageImpl<>(responses, PageRequest.of(0, 20), responses.size());
//                                       데이터 목록    페이지 정보         전체 데이터 개수
```

## ArgumentMatchers (Mockito)

모킹 시 파라미터 조건을 유연하게 지정하는 기능:

```java
// 정확히 이 값일 때만 동작
when(service.getAllPosts(PageRequest.of(0, 20))).thenReturn(...);

// 어떤 Pageable이든 동작 (객체 불일치 방지)
when(service.getAllPosts(any(Pageable.class))).thenReturn(...);
```

| 메서드 | 의미 |
|--------|------|
| `any()` | 아무 값 |
| `any(Class)` | 해당 타입이면 아무 값 |
| `eq(value)` | 정확히 이 값 |
| `anyLong()` | 아무 Long 값 |
| `anyString()` | 아무 String 값 |

# CRUD 메서드별 검증

## Q. 각 CRUD 메서드에 어떤 검증이 필요한가?

**배경:** URL이 `/posts/{postId}/comments/{commentId}`일 때, 검증 없이 commentId만으로 처리하면 다른 게시글에 속한 댓글을 URL 조작으로 삭제/수정할 수 있다.

**A:**

| 메서드 | 필요한 검증 | 이유 |
|--------|------------|------|
| create | 게시글 존재 확인 | 없는 게시글에 댓글 불가 |
| getAll | 게시글 존재 확인 | 없는 게시글이면 빈 리스트가 아니라 404를 줘야 함 |
| get | 댓글 존재 + postId 소속 확인 | URL과 실제 데이터의 일치 보장 |
| update | 댓글 존재 + postId 소속 확인 | 다른 게시글의 댓글을 수정하면 안 됨 |
| delete | 댓글 존재 + postId 소속 확인 | 다른 게시글의 댓글을 삭제하면 안 됨 |

create에서는 postId 소속 확인이 필요 없다 — 댓글이 아직 존재하지 않으니까. 게시글이 실제로 있는지만 확인하면 된다.

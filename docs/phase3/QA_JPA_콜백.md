# JPA 콜백 (@PrePersist, @PreUpdate)

## Q1. 콜백(Callback)이란?

**A:** "나중에 특정 시점이 되면 불러달라"는 패턴이다. 시점은 프레임워크(JPA)가 결정하고, 무엇을 할지는 개발자가 등록한다.

일상 비유: 식당에서 대기표를 받고 "자리 나면 전화해주세요"라고 하는 것. 자리가 나는 시점은 식당이 결정하고, 그때 무엇을 할지(전화)는 내가 미리 등록해둔다.

## Q2. @PreUpdate를 붙이면 어떻게 자동 실행되는가?

**A:** JPA가 UPDATE 쿼리를 실행하기 직전에 `@PreUpdate`가 붙은 메서드를 자동 호출한다.

```
post.setTitle("새 제목"), post.setContent("새 내용")
  → 트랜잭션 커밋
  → JPA: "Post가 변경됐네? UPDATE 해야겠다"
  → JPA: "@PreUpdate가 있네? 먼저 실행" → onUpdate() 호출
  → UPDATE 쿼리 실행 (title, content, updatedAt 모두 반영)
```

**주의:** 콜백 메서드는 JPA가 자동 호출하므로 Service에서 직접 호출하면 2번 실행된다.

## Q3. 한두 줄짜리 코드를 자동화하는 이점은?

**A:** 핵심은 "까먹을 수 없다"는 것이다. 프로젝트가 커지면 Entity를 수정하는 곳이 여러 군데 생긴다 (일반 수정, 상태 변경, 관리자 강제 수정, 배치 처리 등). 수동 방식이면 모든 곳에 시간 갱신 코드를 넣어야 하고, 한 곳이라도 빼먹으면 찾기 어려운 버그가 된다. `@PreUpdate`를 쓰면 어디서 변경하든 JPA가 알아서 호출하므로 누락이 불가능하다.

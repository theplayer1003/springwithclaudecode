# 영속성 컨텍스트 불일치

## Q. 영속성 컨텍스트와 DB 상태가 불일치하면 어떻게 되는가?

**배경:** 이전 CommentService의 deleteComment에서 `post.getComments()`로 댓글 3개를 로딩한 뒤 `deleteById()`로 댓글 1개를 삭제하면, 영속성 컨텍스트의 Post는 여전히 댓글 3개로 알고 있는 불일치가 발생했다.

**A:** 같은 트랜잭션 안에서는 영속성 컨텍스트가 캐시 역할을 한다. DB에서 댓글이 삭제되었어도 영속성 컨텍스트에 이미 로딩된 Post는 여전히 댓글 3개를 갖고 있다. 같은 트랜잭션 내에서 "삭제 후 목록 조회" 같은 로직이 있으면 버그가 된다.

트랜잭션이 끝나면 영속성 컨텍스트가 사라지므로 다음 요청에서는 정상이다.

**해결:** 불필요한 Entity 로딩을 피하면 문제 자체를 원천 차단할 수 있다. `post.getComments()`로 전체를 로딩하지 않고 `commentRepository.findById()`로 댓글만 직접 조회하면 Post가 영속성 컨텍스트에 없으므로 불일치 대상 자체가 없어진다.

# 회원 기능과 권한 적용

---

## 1. Member Entity

사용자 정보를 저장하는 Entity입니다.

```java

@Entity
public class Member {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(unique = true, nullable = false)   // username은 중복 불가
    private String username;

    @Column(nullable = false)
    private String password;                    // BCrypt로 암호화된 비밀번호

    private String email;

    @Enumerated(EnumType.STRING)               // Enum을 문자열로 DB에 저장
    private Role role;                          // USER 또는 ADMIN

    @PrePersist
    public void onCreate() {
        if (this.role == null) {
            this.role = Role.USER;             // 기본 역할은 USER
        }
    }
}
```

### Role Enum

```java
public enum Role {
    USER, ADMIN
}
```

`@Enumerated(EnumType.STRING)`을 사용하면 DB에 `"USER"`, `"ADMIN"` 문자열로 저장됩니다. `EnumType.ORDINAL`(기본값)은 0, 1 같은 숫자로 저장되는데,
Enum에 값이 추가되면 순서가 바뀌어 기존 데이터가 잘못 매핑될 수 있으므로 항상 `STRING`을 사용합니다.

## 2. 회원가입 / 로그인 API

### AuthController

```java

@RestController
@RequestMapping("/auth")
public class AuthController {

    private final AuthService authService;

    @PostMapping("/signup")
    public ResponseEntity<MemberResponse> signup(@Valid @RequestBody SignupRequest request) {
        // 회원가입 처리
    }

    @PostMapping("/login")
    public ResponseEntity<LoginResponse> login(@Valid @RequestBody LoginRequest request) {
        // 로그인 처리 → JWT 토큰 반환
    }
}
```

### AuthService — 회원가입

```java

@Transactional
public MemberResponse signup(SignupRequest request) {
    // 1. username 중복 확인
    if (memberRepository.existsByUsername(request.username())) {
        throw new DuplicateUsernameException("이미 사용 중인 username입니다.");
    }

    // 2. 비밀번호 암호화
    String encodedPassword = passwordEncoder.encode(request.password());

    // 3. Member 생성 및 저장
    Member member = new Member(request.username(), encodedPassword, request.email());
    memberRepository.save(member);

    return MemberResponse.from(member);
}
```

### AuthService — 로그인

```java

@Transactional(readOnly = true)
public LoginResponse login(LoginRequest request) {
    // 1. username으로 사용자 조회
    Member member = memberRepository.findByUsername(request.username())
            .orElseThrow(() -> new AuthenticationFailedException("아이디 또는 비밀번호가 올바르지 않습니다."));

    // 2. 비밀번호 검증
    if (!passwordEncoder.matches(request.password(), member.getPassword())) {
        throw new AuthenticationFailedException("아이디 또는 비밀번호가 올바르지 않습니다.");
    }

    // 3. JWT 토큰 생성
    String token = jwtTokenProvider.createToken(member.getUsername(), member.getRole().name());

    return new LoginResponse(token);
}
```

**보안 포인트:** 로그인 실패 시 "아이디가 틀렸는지" "비밀번호가 틀렸는지"를 구분하지 않습니다. 구분하면 공격자가 유효한 아이디를 먼저 파악한 뒤 비밀번호만 집중 공격할 수 있기 때문입니다.

## 3. 게시글에 작성자 연결

현재 Post Entity에는 작성자 정보가 없습니다. Member와 연관관계를 추가합니다.

```java

@Entity
public class Post {
    // 기존 필드...

    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "member_id")
    private Member member;    // 작성자
}
```

게시글 생성 시 현재 로그인한 사용자를 작성자로 설정합니다:

```java
// PostService
@Transactional
public PostResponse createPost(PostCreateRequest request, String username) {
    Member member = memberRepository.findByUsername(username)
            .orElseThrow(() -> new ResourceNotFoundException("사용자를 찾을 수 없습니다."));

    Post post = new Post(request.title(), request.content(), member);
    postRepository.save(post);

    return PostResponse.from(post);
}
```

Controller에서 현재 로그인한 사용자 정보를 꺼내는 방법:

```java

@PostMapping
public ResponseEntity<PostResponse> createPost(
        @Valid @RequestBody PostCreateRequest request,
        @AuthenticationPrincipal String username) {     // SecurityContext에서 자동 추출
    PostResponse post = postService.createPost(request, username);
    return ResponseEntity.status(HttpStatus.CREATED).body(post);
}
```

`@AuthenticationPrincipal`은 SecurityContext에 저장된 Authentication의 principal을 자동으로 주입합니다. JWT 필터에서 `username`을 principal로
설정했으므로 여기서 그 값을 꺼낼 수 있습니다.

## 4. 작성자만 수정/삭제 가능 (인가)

```java
// PostService
@Transactional
public PostResponse updatePost(Long id, PostUpdateRequest request, String username) {
    Post post = postRepository.findById(id)
            .orElseThrow(() -> new ResourceNotFoundException("게시글을 찾을 수 없습니다."));

    // 작성자 본인인지 확인
    if (!post.getMember().getUsername().equals(username)) {
        throw new AccessDeniedException("본인의 게시글만 수정할 수 있습니다.");
    }

    post.setTitle(request.title());
    post.setContent(request.content());

    return PostResponse.from(post);
}
```

관리자는 모든 글을 수정/삭제할 수 있도록 하려면:

## 5. 전체 구현 요소 정리

### 새로 만들 클래스

| 패키지        | 클래스                             | 역할                   |
|------------|---------------------------------|----------------------|
| entity     | `Member`                        | 사용자 Entity           |
| entity     | `Role` (enum)                   | 사용자 역할 (USER, ADMIN) |
| repository | `MemberRepository`              | 사용자 DB 접근            |
| dto        | `SignupRequest`                 | 회원가입 요청 DTO          |
| dto        | `LoginRequest`                  | 로그인 요청 DTO           |
| dto        | `MemberResponse`                | 회원 응답 DTO            |
| dto        | `LoginResponse`                 | 로그인 응답 DTO (토큰 포함)   |
| service    | `AuthService`                   | 회원가입, 로그인 로직         |
| controller | `AuthController`                | 인증 API 엔드포인트         |
| security   | `JwtTokenProvider`              | JWT 토큰 생성/검증/파싱      |
| security   | `JwtAuthenticationFilter`       | 요청마다 토큰 검증 필터        |
| security   | `SecurityConfig`                | 보안 설정                |
| exception  | `DuplicateUsernameException`    | username 중복 예외       |
| exception  | `AuthenticationFailedException` | 인증 실패 예외             |
| exception  | `AccessDeniedException`         | 권한 없음 예외             |

### 수정할 클래스

| 클래스                      | 변경 내용                         |
|--------------------------|-------------------------------|
| `Post`                   | Member 연관관계 추가                |
| `PostResponse`           | 작성자 정보 포함                     |
| `PostService`            | 작성자 확인 로직 추가                  |
| `PostController`         | `@AuthenticationPrincipal` 사용 |
| `GlobalExceptionHandler` | 새 예외 타입 처리 추가                 |
| `build.gradle.kts`       | Spring Security, JWT 의존성 추가   |
| `application.properties` | JWT 비밀키, 만료 시간 설정             |

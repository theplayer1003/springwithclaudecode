# 라이브러리 의존성 관리

## Q. 라이브러리 주소를 외워야 하는가? 실전에서는 어떻게 찾는가?

**A:** 외우지 않는다. 상황별로 찾는 방법이 다르다.

### 프로젝트 생성 시 — Spring Initializr

start.spring.io에서 필요한 의존성을 검색하여 선택하면 build.gradle에 자동 추가된다.

### 개발 중 추가가 필요할 때 — Maven Central 검색

search.maven.org에서 라이브러리 이름을 검색하면 최신 버전과 Gradle/Maven 의존성 코드가 나온다. 복사해서 붙여넣으면 된다.

### 공식 문서

대부분의 라이브러리는 GitHub README나 공식 문서에 의존성 추가 방법을 안내한다.

## Q. 라이브러리의 기능은 어떻게 파악하는가?

**A:**
- **공식 문서** — 가장 정확한 정보. Spring은 docs.spring.io가 잘 되어 있음
- **IDE 자동완성** — 클래스나 메서드를 입력하면 어떤 기능이 있는지 탐색 가능
- **IDE에서 Ctrl+클릭** — import된 클래스를 클릭하면 소스코드와 JavaDoc을 볼 수 있음

## 참고: 의존성 좌표 형식

`io.jsonwebtoken:jjwt-api:0.12.6`은 `그룹:이름:버전` 형식이다. Maven Central에서 라이브러리를 식별하는 좌표로, 외울 필요 없이 검색해서 복사하면 된다.
